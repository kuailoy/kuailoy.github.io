    # 为什么需要复杂度分析

    讲到数据结构与算法，就一定离不开时间、空间复杂度分析，它是执行效率的一个非常重要的考量指标

    # 事后统计法的问题

    把代码跑一遍，通过统计、监控得到算法执行时间和占用内存大小，这种方法叫事后统计法，但是这种统计方法有很大的局限性：比如测试结果非常依赖测试环境、测试结果受数据规模影响很大

    因此，我们需要一个不需要具体数据测试，就可以粗略估计算法执行效率的方法

    # 大 O 复杂度表示法

    # 方法

    逐行分析代码，假设每个语句的执行 1 遍时间是 1 个 unit_time

    例子 1：

     int cal(int n) {
   int sum = 0;
   int i = 1;
   for (; i <= n; ++i) {
     sum = sum + i;
   }
   return sum;
 }

    第 2、3 行的执行时间为 1 个 unit_time 第 4、5 行的执行时间为 n 个 unit_time 因此总执行时间为(2n+2) * unit_time

    例子 2：

     int cal(int n) {
   int sum = 0;
   int i = 1;
   int j = 1;
   for (; i <= n; ++i) {
     j = 1;
     for (; j <= n; ++j) {
       sum = sum +  i * j;
     }
   }
 }

    第 2、3、4 行的执行时间为 1 个 unit_time 第 5、6 行的执行时间为 n 个 unit_time 第 7、8 行的执行时间为 n^2 个 unit_time 因此总执行时间为(2n^2+2n+3) * unit_time

    综上，可以得出一个结论，所有代码的执行时间 T(n) 与每行代码的执行次数 n 成正比。

    # 公式

    • T(n)表示代码的执行时间
    • n 表示数据规模的大小
    • f(n)表示每行代码执行次数总和
    • O 表示执行时间与 f(n)表达式成正比

    # 时间复杂度分析

    实际案例中，运用下列方法进行时间复杂度分析

    # 1.只关注循环执行次数最多的一段代码

    在例子 1 中，2、3 行为常量级执行时间，可以忽略不计，只需关注 4、5 行，这两行被执行了 n 次，因此总的时间复杂度为 O(n)

    # 2.加法法则：总复杂度等于量级最大的那段代码的复杂度

    如果 那么

    例子 3：

    int cal(int n) {
  // 常量级，可以忽略
  int sum_1 = 0;
  int p = 1;
  for (; p < 100; ++p) {
    sum_1 = sum_1 + p;
  }

  // O(n)
  int sum_2 = 0;
  int q = 1;
  for (; q < n; ++q) {
    sum_2 = sum_2 + q;
  }

  // O(n^2)
  int sum_3 = 0;
  int i = 1;
  int j = 1;
  for (; i <= n; ++i) {
    j = 1;
    for (; j <= n; ++j) {
      sum_3 = sum_3 +  i * j;
    }
  }

  return sum_1 + sum_2 + sum_3;
}

    分析：第一段代码为常量级，第二段时间复杂度为 O(n)，第三段时间复杂度为 O(n^2)

    根据加法原则，总复杂度等于量级最大的那段代码的复杂度，因此整段代码的时间复杂度为 O(n^2)

    # 空间复杂度分析

    时间复杂度的全称是渐进时间复杂度，表示算法的执行时间与数据规模之间的增长关系 空间复杂度的全称是渐进空间复杂度（asymptotic space comlexity），表示算法的储存空间与数据规模之间的增长关系

    例子 8：

    
void print(int n) {
  int i = 0;
  int[] a = new int[n];
  for (i; i <n; ++i) {
    a[i] = i * i;
  }

  for (i = n-1; i >= 0; --i) {
    print out a[i]
  }
}

    上面的代码中，第 2 行申请了一个存储变量 i ，因为它是常量阶的，跟数据规模 n 没有关系 第 3 行申请了一个大小为 n 的 int 类型数组，剩下的代码没有占用更多的空间，所以整段代码的空间复杂度是 O(n)

    常见的空间复杂度就是，像这样的对数阶复杂度平时都用不到
