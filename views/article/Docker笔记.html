    # 查看docker版本信息
docker version

    # Docker Image

    • 文件和meta data 的集合（root filesystem）
    • 分层的，并且每一层都可以添加改变删除文件，成为一个新的Image
    • 不同的Image可以共享相同的layer
    • Image本身是read-only的
    # 查看本地安装的image
$ docker image ls
#等同于
$ docker images

# Pull from Registry
docker pull ubuntu:14.04

# 删除 image 文件
$ docker image rm [imageName]
# 等同于
$ docker rmi [imageID]

    # Docker Container

    image 文件生成的容器实例，本身也是一个文件，称为容器文件**。**也就是说，一旦容器生成，就会同时存在两个文件： image 文件和容器文件。而且关闭容器并不会删除容器文件，只是容器停止运行而已

    • 通过Image创建（copy）
    • 在Image layer之上建立一个container layer（可读写）
    • 类比面向对象：类和实例
    • Image负责app的存储和分发，Container负责运行app
    
# 列出本机正在运行的容器
$ docker container ls

# 列出本机所有容器，包括终止运行的容器
$ docker container ls --all
# 等同于
$ docker ps -a

# 运行image
$ docker container run hello-world

# 终止容器
$ docker container kill [containID]

# 删除容器
$ docker container rm [containerID]
# 等同于
$ docker rm [containerID]

# 列出所有容器ID
$ docker container ls -aq

# 删除所有容器
$ docker rm $(docker container ls -aq)

# 列出所有退出容器ID
$ docker container ls -f "status=exited" -q

# 删除所有退出容器
$ docker rm $(docker container ls -f "status=exited" -q)

    常用的方法有两种

# 用下面的命令查看image layer
$ docker history [imageId]

    可以看到，生成container的image和新的image有很多层是共享的，只是在原基础上加了一层

    # 2. 基于Dockerfile

    编写dockerfile/.dockerignore .dockerignore 示例

    
.git
node_modules
npm-debug.log

    dockerfile

    
FROM node:8.4
COPY . /app
WORKDIR /app
RUN ["npm", "install"]
EXPOSE 3000/tcp

# CMD表示容器启动后自动执行的命令，指定了CMD命令以后，docker container run命令就不能附加命令了
CMD node demos/01.js

    创建image文件

    # -t 指定image名称，冒号后面跟版本号，不指定的话就是latest
# 最后的点表示dockerfile文件的路径
$ docker image build -t koa-demo .
# 或者
$ docker image build -t koa-demo:0.0.1 .

    创建成功后，查看image就可以看到新生成的image

    $ docker image ls

    # 生成容器

    
$ docker container run -p 8000:3000 -it koa-demo /bin/bash
# 或者
$ docker container run -p 8000:3000 -it koa-demo:0.0.1 /bin/bash
    • -p参数：容器的 3000 端口映射到本机的 8000 端口。
    • -it参数：容器的 Shell 映射到当前的 Shell，然后你在本机窗口输入的命令，就会传入容器。
    • koa-demo:0.0.1：image 文件的名字（如果有标签，还需要提供标签，默认是 latest 标签）。
    • /bin/bash：容器启动以后，内部第一个执行的命令。这里是启动 Bash，保证用户可以使用 Shell。

    # 终止容器

    在容器的命令行，按下 Ctrl + c 停止 Node 进程（如果有），然后按下 Ctrl + d （或者输入 exit）退出容器 也可以用 docker container kill 终止容器运行。容器停止运行之后，并不会消失，使用 docker container rm 命令删除容器文件。

    # 在本机的另一个终端窗口，查出容器的 ID
$ docker container ls

# 停止指定的容器运行
$ docker container kill [containerID]

# 查出容器的 ID
$ docker container ls --all

# 删除指定的容器文件(id不需要写全)
$ docker container rm [containerID]

    # CMD and ENTRYPOINT

    • RUN: 执行命令并创建新的Image Layer
    • CMD:
      • 容器启动后默认执行的命令和参数
      • 如果docker run 指定了其他命令 CMD命令被忽略
      • 如果定义了多个CMD，只有最后一个会执行
    • ENTRYPOINT：
      • 让容器以应用程序或者服务的形式运行（比如数据库服务）
      • 不会被忽略，一定会执行
      • 最佳实践：写一个shell脚本作为entrypoint

    Shell格式

    RUN apt-get install -y vim
CMD echo "hello docker"
ENTRYPOINT echo "hello docker"

    Exec格式

    RUN ["apt-get", "install", "-y", "vim"]
CMD ["/bin/echo", "hello docker"]
ENTRYPOINT ["bin/echo", "hello docker"]
