    MacOS终端神器zsh配置(oh-my-zsh)

    kuailoy 12/5/2019 linuxmacOS

    mac 系统下默认的shell应该是bash，个人感觉还算好用，但是看到很多人在推荐oh-my-zsh，忍不住又想折腾一下了，还是遇到了一些小坑， 这里记录一下

    系统： macOS Catalina

    # 使用iTerm2

    如果没有安装iTerm2，请自行安装

    # 使用zsh

    Linux/Unix系统提供了很多shell，输入命令查看当前可用的shell

    cat /etc/shells

    我的结果如下：

    /bin/bash
/bin/csh
/bin/dash
/bin/ksh
/bin/sh
/bin/tcsh
/bin/zsh

    可以看到Mac自带了zsh, 可以使用Homebrew更新为最新版

    # macOS系统安装zsh(更新)
brew install -without-etcdir zsh
# 添加zsh shell路径至/etc/shells文件中，复制"/usr/local/bin/zsh"到/etc/shells中
sudo echo '/usr/local/bin/zsh' >> /etc/shells

    更换默认shell为zsh

    chsh -s /usr/local/bin/zsh

    # 安装oh-my-zsh

    使用官方提供的命令安装

    通过curl

    sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

    通过wget

    sh -c "$(wget -O- https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

    如果安装失败，可以试试下面的

    自动安装：（老链接，会重定向到最新的路径，如果上面的失败，这个可能也会失败）

    wget https://github.com/robbyrussell/oh-my-zsh/raw/master/tools/install.sh -O - | sh

    手动安装：（我使用这种方法安装成功）

    git clone git://github.com/robbyrussell/oh-my-zsh.git ~/.oh-my-zsh
cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

    安装成功后重启iTerm2，就可以看到彩色提示了

    # zsh配置

    zsh的配置文件在~/.zshrc下面，可以按照自己的需求进行配置, 下面列出几个常用的配置：

    # 更换主题

    在~/.oh-my-zsh/themes目录下提供了各式各样的主题文件，可以通过命令查看

    ls ~/.oh-my-zsh/themes

    在使用这些好看的主题前要确保安装了powerline-font (opens new window), 否则有些字符可能会展示不出来

    # clone
git clone https://github.com/powerline/fonts.git --depth=1
# 安装powerline-font
cd fonts
./install.sh
# 清除安装目录
cd ..
rm -rf fonts

    重启iTerm2，在Preferences -> Profiles -> Text中将字体设置为powerline-font相关字体，我用的是Source Code Pro for Powerline

    然后就可以更换主题了！以agnoster (opens new window)这个主题为例：

    可以先检测下该主题下所需要的图标是否能正常显示

    # 图标检测
echo "\ue0b0 \u00b1 \ue0a0 \u27a6 \u2718 \u26a1 \u2699"

    正常显示如下图

    修改配置文件， 使用编辑器打开~/.zshrc, 找到ZSH_THEME，修改为agnoster（默认值为robbyrussell)

    ZSH_THEME="agnoster"

    保存退出，重启iTerm2， 效果如下

    设置随机主题，每次打开zsh都会随机加载一个主题

    ZSH_THEME = "random"

    发现喜欢的主题，使用这条命令查看当前主题名称

    echo $RANDOM_THEME

    再修改为当前名称的主题就可以了

    其实随机主题一般会在打开终端的时候显示在头部

    # 常用插件

    在 ~/.zshrc 中配置

    plugins=(git 其他插件)

    将需要用到的插件安装后，写在上面的括号里，用空格隔开

    # git plugin (opens new window)

    oh-my-zsh默认配置了git插件，提供了git操作的快捷键，显示当前分支信息等

    例如：gcl -> git clone

    完整列表

    # 便捷目录切换 autojump (opens new window)

    mac使用Homebrew安装

    brew install autojump

    然后将下列代码加入配置文件~/.zshrc

    [[ -s ~/.autojump/etc/profile.d/autojump.sh ]] && . ~/.autojump/etc/profile.d/autojump.sh

    安装了autojump之后，zsh 会记录你访问过的路径, 实现无视当前路径级别的快速跳转，还支持模糊匹配和自动补全

    # 快捷跳转目录
j dir
# 通过finder打开目录
jo dir
# 查看历史路径库
j -s

    例如想要跳转到~/.oh-my-zsh，直接输入命令 j oh 即可

    # 自动补全 zsh-autosuggestions (opens new window)

    根据历史记录补全输入内容

    安装

    git clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

    修改~/.zshrc

    plugins=([plugins...] zsh-autosuggestions)

    注意：如果没有看到灰色的自动补全提示，但是可以用 -> 自动补全，可能是iTerm的配色方案导致的，要确保"ANSI Normal Black"和"Background"设置为不同颜色

    iTerm > Preferences > Profiles > Colors > ANSI Colors > Bright > Black iTerm > Preferences > Profiles > Colors > Basic Colors > Background

    我在使用iTerm2的预设配色方案Solorized Dark就遇到了这个问题

    # 语法高亮显示 zsh-syntax-highlighting (opens new window)

    安装

    git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

    修改~/.zshrc

    plugins=([plugins...] zsh-syntax-highlighting)
