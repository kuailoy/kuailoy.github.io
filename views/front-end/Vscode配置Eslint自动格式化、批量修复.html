    Vscode配置Eslint自动格式化、批量修复

    kuailoy 8/20/2021 vscode 踩坑

    # 开启Eslint

    1. 安装eslint插件
    2. 安装eslint npm包及相关插件

    # 保存时根据eslint配置自动格式化

    配置vscode的 settings.json 文件

    "eslint.validate": [
  "javascript",
  "javascriptreact",
  "html",
  "vue"
],
"editor.codeActionsOnSave": {
  "source.fixAll.eslint": true
},

    # 批量修复

    在 package.json 中，增加 scripts

    "scripts": {
		// ...
    "lint": "eslint src --ext .js,.jsx",
    "lint:fix": "eslint src --fix --ext .js,.jsx"
  },

    ​npm run lint:fix 改操作会修复所有能够被自动修复的问题，建议操作后在git change中浏览一遍代码

    # Eslint配置错误排查

    如果Eslint有问题，例如不能正确显示报错红线，可以检查vscode的output面板 例如上图中，可以看出是 max-line-length 这项配置有问题
