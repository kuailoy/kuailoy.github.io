    TypeScript学习笔记4 - 接口

    kuailoy 8/26/2019 typescript

    # 接口的应用场景

    • 对象的形状（Shape）的描述

    • 对类的部分行为进行抽象

    这里先说说第一种用法

    // 简单的例子
interface Person {
  name: string;
  age: number;
}

let tom: Person = {
  name: 'Tom',
  age: 18
}

    接口命名时，首字母一般大写， 有些规范会要求加上I前缀，例如IComponent

    赋值时，变量形状必须与接口形状一致，缺少属性或多出属性都是不允许的，如下所示：

    // 接上面的例子
let jack: Person = {
  name: 'Jack'  // error: 缺少必要的属性'age'
}

let susan: Person = {
  name: 'Susan',
  age: 20,
  gender: 'female'  // error: 'gender'属性在类型'Person'上不存在
}

    # 可选属性

    有时我们并不希望完全匹配一个形状，可以添加可选属性，该属性可以不存在

    interface Person {
  name: string;
  age: number;
  gender?: string
}

let jack: Person = {
  name: 'Jack',
  age: 25
}   // ok

let susan: Person = {
  name: 'Susan',
  age: 20,
  gender: 'female'
}   // ok

let tracy: Person = {
  name: 'Tony',
  age: 35,
  gender: 'male',
  lastname: 'Stark' // error: 'lastname'属性在类型'Person'上不存在
}

    仍然不允许添加未定义属性

    # 绕过未定义属性检查

    有时我们不希望TypeScript对接口属性做严格的数据检查，比如只有Person的name是必要属性，其他属性可有可无，并且我们不关心其他属性的类型， 可以用下列方法：

    # 1. 使用类型断言

    interface Person {
  name: string;
}
let jack: Person = {
  name: 'Jack',
  age: 18
} as Person

    # 2. 定义索引属性

    interface Person {
  name: string;
  age?: number;
  [propName: string]: any;
}
let jack: Person = {
  name: 'Jack',
  gender: 'male
}

    一旦定义了索引属性，当前确定属性、可选属性必须是该类型的子集

    interface Person {
  name: string;
  age?: number;   // error：属性'age'的类型'number | undefined'不能指定给索引类型'string'
  [propName: string]: string;
}

    可修改为

    interface Person {
  name: string;
  age?: number;
  [propName: string]: string | number | undefined;  // 或者直接用any
}

    另外，索引属性的属性名类型为string时，即使属性名类型为number也可通过，因为在对象中，属性名会被转成string类型

    interface Food {
  [propName: string]: string
}
const fruit: Food = {
  0: 'Apple',
  1: 'Banana',
}  // ok

    索引属性的属性名类型也可以直接设置为number，如下：

    interface Food {
  [id: number]: string
}
const fruit: Food = {
  0: 'Apple',
  1: 'Banana',
}
const vegetables: Food = {
  a: 'Carrot',   // error: 属性'a'在类型'Food'中不存在
  '2': 'Cabbage' // ok 数值作为属性名会最终转成字符串，所以这里也符合接口定义
}

    # 3. 利用类型兼容性

    interface Person {
  name: string;
}

let jack: Person;
let j = {
  name: 'Jack',
  age: 18
}
jack = j;  // ok

    类型兼容：如果要将b赋值给a， b要包括a的所有属性，多余的无所谓

    在上例中，j包含属性name, 兼容类型Person，所以可以赋值给jack

    类型兼容这种方法不推荐使用，其原因我认为是存在隐式的类型变化，容易导致混乱，况且，如果变量j没有其他用途，就白白增加了一个变量

    而反观前两种，无论是类型断言还是索引属性，都明确表示了可能存在的类型变化

    # 只读属性

    设置一个接口的属性为只读，只需要在属性名前面加上readonly关键字

    interface Person {
  name: string;
  age?: number;
  readonly gender: string   // 只读属性
}

let jack: Person = {
  name: 'Jack',
  gender: 'male'
}

jack.gender = 'female'; // error: 不能给只读属性'gender'赋值

    # 函数类型

    接口还可以描述函数类型，函数的参数类型、个数，返回值类型都要与接口一致

    interface AddFunc {
  (a: number, b: number): number
}

const add: AddFunc = (num1, num2) => num1 + num2;
const join: AddFunc = (m, n) => `${m}${n}`; // error: 不能将'string'分配给类型'number'
add(1, '2');    // error: 不能将参数类型'2'分配给类型'number'

    # 接口继承

    通过接口继承可以提高接口的复用性

    interface Food {
  type: string
}
interface Fruit {
  color: string
}
interface Apple extends Fruit {  // 继承
  radius: number
}
interface Banana extends Fruit, Food {  // 可以继承多个接口
  length: number
}
const apple: Apple = {
  color: 'red',
  radius: 3,
};
const banana: Banana = {
  type: 'fruit',
  length: 20
};    // error: 缺少属性'color'

    # 混合类型接口

    在TypeScript中函数也可以有属性，我们可以用混合类型接口定义函数属性类型

    interface Counter {
  (): void,
  count: number
}

const getCounter = (): Counter => {
  const c = () => {
    c.count++;
  }
  c.count = 0
  return c;
}

const counter = getCounter();
counter();
console.log(counter.count) // 1
counter.count = 'hello'   // error: 类型'hello'不能赋值给类型'number'
