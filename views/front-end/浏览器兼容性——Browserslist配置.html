    浏览器兼容性——Browserslist配置

    kuailoy 6/8/2020 browserwebpack

    browserslist (opens new window) 是一个前端构建工具公用的配置项，可以指定构建后的代码所兼容的目标浏览器版本或 node 版本 它可以被以下工具共同使用：

    • Autoprefixer (opens new window)
    • Babel (opens new window)
    • post-preset-env (opens new window)
    • eslint-plugin-compat (opens new window)
    • stylelint-unsupported-browser-features (opens new window)
    • postcss-normalize (opens new window)
    • ...

    # 配置

    有两种方式配置 browserslist

    1. 在 package.json 增加字段
    {
  "browserslist": ["last 1 version", "> 1%", "maintained node versions", "not dead"]
}
    1. 在项目根目录下新建 .browserslistrc
    last 1 version
> 1%
maintained node versions
not dead

    配置项的详细说明请参考browserslist 文档 (opens new window)，这里就不搬运了

    # 查看配置

    在工程目录下运行  npx browserslist  来查看你配置的浏览器版本范围

    # 案例

    最近在做一个项目要求兼容 IE9 和较早版本的浏览器，使用的配置如下

    {
  "browserslist": ["last 2 version", "> 0.2%"]
}
