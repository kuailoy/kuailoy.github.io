        TypeScript学习笔记6 - 泛型

        kuailoy 2019-08-28 typescript

        泛型（Generics）是指在定义函数、接口或类的时候，不预先指定具体的类型，而在使用的时候再指定类型的一种特性。

        # 使用any的问题

        现在有一个函数createArray， 它可以创建一个指定长度的数组并使用传入的值来填充：

        function createArray(length: number, value: any): any[] {
  return new Array(length).fill(value);
}
createArray(3, 'hello');  // ['hello', 'hello', 'hello']

        显然，该函数返回值中熟组的每一项应该与参数value具有相同的类型

        但由于我们使用了any，TypeScript不会对返回值数组内的元素进行类型检查

        考虑下面的情况：

        let arr = createArray(3, 1);  // [1, 1, 1]
arr.forEach(item => console.log(item.length)); // item是number类型，没有length属性，但这里没有报错

        这时泛型就派上用场了

        # 使用泛型

        function createArray<T>(length: number, value: T): T[] {
  return new Array(length).fill(value);
}
let arr1 = createArray<number[]>(3, [1]);  // 使用时指定T类型'number[]'   [[1], [1], [1]]
arr1.forEach(item => console.log(item.length)); // ok

let arr2 = createArray<number>(3, 1);  // 使用时指定T类型'number'  [1, 1, 1]
arr2.forEach(item => console.log(item.length)); // error: 类型'number'不存在属性'length'

        在声明函数时，函数名后用<T>定义泛型，其中T代表输入的类型，返回值T[]中元素类型与T一致

        在调用时指定T的类型，TypeScript就能进行类型检查了，如果不手动指定类型，TypeScript会自动推算出来

        let arr2 = createArray(3, 1);  // TypeScript推算出T类型为'number'  [1, 1, 1]
arr2.forEach(item => console.log(item.length)); // error: 类型'number'不存在属性'length'

        # 定义多个泛型

        function swap<T, U>(tuple: [T, U]): [U, T] {
  return [tuple[1], tuple[0]]
}
swap([7, 'seven']); // ['seven', 7]

        # 泛型约束

        在函数内部使用泛型的时候，由于不知道泛型的类型，所以不能随意使用其属性

        function loggingIdentity<T>(arg: T): T {
  console.log(arg.length);  // error: 类型'T'上不存在属性'length'
  return arg;
}

        # 属性约束

        这时我们可以通过一个类型和extends对泛型进行约束

        interface Lengthwise {
  length: number
}
// 使用extends约束'T'类型上必须有'length'属性
function loggingIdentity<T extends Lengthwise>(arg: T): T {
  console.log(arg.length);
  return arg;
}

        如果我们传入的参数上没有length属性，那么编译阶段就会报错了

        loggingIdentity('abc');  // ok
loggingIdentity([1, 2, 3]);  // ok
loggingIdentity(7);  // error: 参数类型'7'不能指定为类型'Lengthwise'

        # 参数之间的相互约束

        function copyFields<T extends U, U>(target: T, source: U): T {
  for (let key in source) {
    target[key] = (<T>source)[key]
  }
  return target;
}

let obj = { a: 1, b: 2, c: 3, d: 4 };
copyFields(obj, { b: 10, d: 20 });

        上面的代码中，使用T 继承U， 保证了U上面不会出现T没有的字段

        # 属性名约束

        function getValue<T, K extends keyof T>(object: T, key: K) {
  return object[key]
}
const obj = { a: 1, b: 2 };
getValue(obj, 'c');   // error: 类型'"c"'的参数不能赋值给类型'"a" | "b"'

        这里使用K继承的keyof T，可以看作是T的key值组成的联合类型
