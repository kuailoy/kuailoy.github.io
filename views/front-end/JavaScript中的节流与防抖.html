    JavaScript中的节流与防抖

    kuailoy 5/24/2020 javascript

    在前端开发中，经常需要监听处理一些高频率事件，例如调整窗口大小(resize)，页面滚动(scroll)，用户输入(input)，鼠标移动(mousemove)等，为了提高性能，防止浏览器卡死、服务器爆炸，我们需要用到防抖函数或节流函数

    # 节流函数(throttle)

    概念： 在单位时间段内多次调用函数，最多只能触发一次。

    原理： 限制高频率事件的执行频率，好比将水龙头关小，让水一滴一滴的流

    例如：设置单位时间为 1000ms，追踪用户的鼠标移动，如果用户在 10 秒(10000ms)内不停地移动鼠标，节流之前，事件会被触发成百上千次，节流之后，每 1000ms 只会触发一次，其他的事件触发会被忽略，因此理论上只会触发 10 次。

    下面我们以 mousemove 事件为例演示

    首先看下高频率事件的触发，我们来画一个html

    <!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta http-equiv="X-UA-Compatible" content="ie=edge" />
    <title>throttle</title>
  </head>
  <style>
    #box {
      width: 100%;
      height: 200px;
      line-height: 200px;
      text-align: center;
      background-color: lightblue;
      font-size: 30px;
    }
  </style>
  <body>
    <div id="box"></div>
    <script src="throttle.js"></script>
  </body>
</html>

    throttle.js代码如下

    var count = 1;
var box = document.getElementById('box');

function move() {
  box.innerHTML = count++;
}

box.onmousemove = move;

    这里可以看到事件可怕的触发频率，如果事件处理函数稍微复杂一点，或者我们需要发送ajax请求，结果可想而知...

    # 简化版 throttle

    用时间戳的方法实现节流，非常简单！

    // 简化版
function throttle(func, wait) {
  var previous = 0;
  return function() {
    // 获取当前时间戳
    var now = Date.now(),
      remaining = wait - (now - previous);
    // 时间间隔超出wait，执行
    if (remaining <= 0) {
      // 更新初始时间
      previous = now;
      func();
    }
  };
}
box.onmousemove = throttle(move, 1000);

    throttle 函数通过比较当前和上次执行的时间差来节流，只有当两次执行的时间差超过预设值 1000ms，才会触发真正的事件回调函数，效果如下：

    可以看出，这个函数已经实现了节流的功能，但还存在一些问题

    # 事件回调函数的this

    在 move 方法中作如下修改：

    function move() {
  console.log(this);
  box.innerHTML = count++;
}

    在使用throttle之前，this指向<div id="box"></div>, 而使用throttle之后this却指向了window！ 这是因为move被包裹在节流后的函数里执行，而执行时的上下文是window

    # 事件回调函数的 event 对象

    不使用throttle时，将 move 作为事件回调函数，可以轻松获取到 event 对象，以进行相关的操作

    function move(e) {
  console.log(e);
  box.innerHTML = count++;
}

    节流后，move 被包裹在外层的事件函数中执行，event 对象丢失

    # 基本款 throttle

    为了解决 this 和 event 对象丢失的问题，对 throttle 作如下修改

    // 基本款
function throttle(func, wait) {
  var context, args;
  // 初始时间戳
  var previous = 0;
  return function() {
    // 缓存this, arguments
    context = this;
    args = arguments;
    // 获取当前时间戳
    var now = Date.now(),
      remaining = wait - (now - previous);
    // 时间间隔超出wait，执行
    if (remaining <= 0) {
      // 更新初始时间
      previous = now;
      func.apply(context, args);
    }
  };
}

    使用apply将 this 和 event 传给事件函数

    到这里一个基本款的 throttle 函数完成，可以应用于一些简单的需求啦。

    # 定时器版 throttle

    使用定时器来实现 throttle 也是一种常见的做法

    function throttle(func, wait) {
  var timeout, context, args;
  return function() {
    context = this;
    args = arguments;

    if (!timeout) {
      timeout = setTimeout(function() {
        timeout = null;
        func.apply(context, args);
      }, wait);
    }
  };
}

    # 首次和末尾执行

    在使用 throttle 时，考虑下面两个问题：

    • 第一次触发时，是否立即执行？
    • 最后一次触发完成，等待时间结束后，是否会再执行一次？

    通过观察，可以发现，时间戳版的throttle首次触发会立即执行，最后不会再执行一次，因为函数内进行的是前置条件判断； 而定时器版的throttle刚好相反，首次不执行，末尾多执行一次，因为用户停止操作后，定时器读秒结束还会再执行一次

    如果想要想要通过一个函数，控制首尾是否执行，以满足不同的需求，可以将这两种实现方式加以结合，参考underscore中的实现

    # underscore 中的 throttle 的实现

    // option为可选参数，不设置时，首次立即执行并且末尾多执行一次
// 首次不执行， option为 { leading: false }
// 末尾不执行，option为 { trailing: false }
function throttle(func, wait, option) {
  var timeout, context, args, result;
  var previous = 0;
  // 可选参数给默认值
  option = option || {};

  var later = function() {
    previous = option.leading === false ? 0 : Date.now();
    timeout = null;
    result = func.apply(context, args);
    if (!timeout) context = args = null;
  };

  var throttled = function() {
    var now = Date.now();
    // 第一次执行前进行判断
    if (!previous && option.leading === false) previous = now;
    // 计算当次执行与上次执行的时间差
    var remaining = wait - (now - previous);
    context = this;
    args = arguments;
    // 到达了预设时间，或者修改了客户端时间
    // 因为(now - previous)一定为正数，所以remaining不会大于wait，除非修改了时间
    if (remaining <= 0 || remaining > wait) {
      // 清除定时器，确保func不会重复执行
      if (timeout) {
        clearTimeout(timeout);
        timeout = null;
      }
      // 更新上次执行时间
      previous = now;
      // 执行函数
      result = func.apply(context, args);
      // 释放内存，时间戳逻辑结束
      if (!timeout) context = args = null;
    } else if (!timeout && option.trailing !== false) {
      // 未设置末尾不执行，剩余时间结束后，总会执行一次，注意使用不全等符号避免类型转换
      timeout = setTimeout(later, remaining);
    }
    // 返回func的返回值
    return result;
  };
  return throttled;
}

    补充说明：

    1. 再来看下later函数

      var later = function() {
  previous = option.leading === false ? 0 : Date.now();
  // ...
};

      注意这行代码在 leading 为 false 时把 previous 置为 0，这样才会再接下来执行throttled中if (!previous && option.leading === false) previous = now;，previous的值又变成了now，控制了首次不会立即执行，相当于又回到了初始状态

    2. throttle函数中的result作用是保存传入函数的返回值，避免原函数的返回值丢失，但是放在定时器外面，只能保存上一次的返回值(真的有意义么...)

    3. 如果设置了trailing:false，无疑只会走时间戳的逻辑，如果没有设置呢？当remaing到达 0，时间戳和定时器哪个会先执行？大部分时候会走定时器的逻辑，但是也可能走时间戳，因此我们要在时间戳中清除定时器，避免重复执行func。从这里可以看出，时间戳和定时器互为补充，从而控制首次和末尾的执行

    4. 不能同时设置leading: false和trailing: false，否则 leading:false 会失效，因为设置了 trailing:false,不能走定时器逻辑，就变成了纯时间戳的版本，也就是在第一次执行后，以后首次触发都会立即执行了

    # 取消节流

    在 underscore 的 throttle 函数中，还提供了这样一个方法，用来取消当次节流，调用后立刻回到初始状态

    function throttle(func, wait, option) {
  // ...
  var throttled = function() {
    // ...
  };
  throttled.cancel = function() {
    clearTimeout(timeout);
    previous = 0;
    context = args = null;
  };
  return throttled;
}

    到这里，就是一个完整的 underscore 版的 throttle 了～

    # 防抖函数(debounce)

    概念： 函数被持续调用时不会触发，停止调用 n 毫秒后才会触发一次

    例如：搜索框 input 事件，要实现自动搜索时，可以使用防抖，输入停顿 500ms 后理解为输入完成，才发请求，从而减少开支，又如浏览器 resize 时，只取最后一次调整结果，也可在停顿一定时间后视为调整完成，再去触发回调函数

    这里还用上面的 mousemove 事件做演示

    # 简化版 debounce

    function debounce(func, wait) {
  var timeout;
  return function() {
    clearTimeout(timeout);
    timeout = setTimeout(func, wait);
  };
}

box.onmousemove = debounce(move, 1000);

    效果如下:

    原理很简单，通过定时器，将获取到的用户操作延迟处理，如果等待期间再次触发，则清除上一个定时器，重新计时

    跟上面的throttle类似，这里的this和event对象丢失，需要补回来

    # 基本款 debounce

    function debounce(func, wait) {
  var timeout, context, args;
  return function() {
    var context = this;
    var args = arguments;

    clearTimeout(timeout);
    timeout = setTimeout(function() {
      func.apply(context, args);
    }, wait);
  };
}

    到这里，debounce 的基本功能实现

    # 立即执行

    观察上面例子，可以发现move每次都是延迟执行，如果我们想要先立即执行一次，再去限制停顿 n 毫秒秒后才能执行呢，也就是改后置执行为前置执行。这里仍然参照 undercore 的实现做下修改

    // underscore源码简化版，实现原理一致
// immediate 可选参数，传true立即执行
function debounce(func, wait, immediate) {
  // 与上面的throttle一样，result用来保存原函数的返回值
  var timeout, result;
  // 防抖处理后的函数
  var debounced = function() {
    // 缓存this和event对象
    var context = this;
    var args = arguments;
    // 实现防抖的关键一步！清除定时器
    if (timeout) clearTimeout(timeout);
    // 是否立即执行
    if (immediate) {
      // 根据timeout是否存在判断是否立即执行
      var callNow = !timeout;
      // 等待事件过后，将timeout置为null，下次执行时callNow为true
      timeout = setTimeout(function() {
        timeout = null;
      }, wait);
      if (callNow) result = func.apply(context, args);
    } else {
      // 没有传递immediate参数，和上面基本款逻辑一致
      timeout = setTimeout(function() {
        result = func.apply(context, args);
      }, wait);
    }
    return result;
  };
  return debounced;
}

    然后修改一下 debounce 的调用

    box.onmousemove = debounce(move, 1000, true);

    效果如下：

    # 取消防抖

    最后，和 throttle 函数一样， 给 debounce 添加一个 cancel 方法，用于取消防抖的等待时间

    function debounce(func, wait, immediate) {
  // ...
  debounced.cancel = function() {
    clearTimeout(timeout);
    timeout = null;
  };
  return debounced;
}

    至此，underscore 版的防抖函数也实现完成～
