    JavaScript 高级程序设计

    kuailoy 2/2/2021 javascript

    # 前言

    《JavaScript 高级程序设计》这本书，我一直都没读完，之前粗略的读了一部分，每次想拿起来，都对它的厚度望而生畏。阅读大部分技术书的时候，我经常是基础的部分看完了，进阶的部分有点吃力，就放弃了，这样只是对现有的知识做了复习巩固，并没有学到太多新东西，学到的东西也会很快忘掉。

    因此笔记是读书的关键，尤其是这种比较厚的书，完整的读下来，既耗时，又抓不住关键。另外，这本书还有很大一部分在讲兼容性和历史遗留问题，这部分可以忽略掉，太过基础的地方也可以忽略掉。这次读，我决定按章节来，每个章节读完，记下关键的知识，或者之前自己不了解的知识，这样还可以经常看看笔记来回顾一下。

    这本书的第四版已经出来了，有一些 ES6 相关的内容补充，我这里因为已经买过第三版了，就不再买第四版了，精华部分应该是一样的。

    这就开始吧。

    # 第一章 JavaScript 简介

    JavaScript 的历史，略过 JavaScript 的实现由三部分组成

    • ECMAScript，由 ECMA-262 定义，提供核心语言功能
    • 文档对象模型（DOM），提供访问和操作网页内容的方法和接口
    • 浏览器对象模型（BOM），提供与浏览器交互的方法和接口

    ECMAScript 的几个重要版本：3、3.1、5、6 浏览器对 JavaScript 语言的支持，略过

    # 第二章 在 HTML 中使用 JavaScript

    # 使用 script 标签的两种方式：

    • 页面中嵌入 JavaScript 代码
    • 加载外部 JavaScript 文件

    # script 标签的几个属性

    • src：引入外部文件的 url，可跨域
    • type：一般为 text/javascript，不设置时默认为 text/javascript
    • async：异步加载 script，不阻塞文档，只在引用外部文件时生效
    • defer：异步加载 script，延迟到文档加载完之后执行，只在引用外部文件时生效

    # async 和 defer

    首先，不设置 async 和 defer 的情况下，script 标签是同步加载的，如果它放在 html 的中间部分，加载到这个 script 时，就会阻塞后面 html 文档的加载，当 script 内的代码执行完毕后，再继续加载剩余的 html。

    如果设置了 async 属性，script 标签开始加载时，是不阻塞后面的 html 加载的，但是当 script 标签加载完后，不论 html 是否加载完，浏览器都会暂停对 html 的解析，先执行当前 script 里面的 javascript。 所以 async 是不可控的，如果某个 script 存在 dom 操作，是不能保证获取到 dom 的，多个 async 的执行顺序也不能确定，如果 script 存在相互依赖关系，就不要使用 async 了。

    如果设置了 defer 属性，script 会立即加载，不阻塞 html，并且会等待 html 文档加载完毕在执行，多个 defer 标签也会按顺序执行（但是现实中并不一定会按顺序执行）。

    出于兼容性和可控性考虑，把 script 标签放在页面底部仍是最佳选择。

    # <noscript>元素

    当浏览器不支持 JavaScript，或者 JavaScript 被禁用时，才会展示<noscript>中的内容，作为对用户的提示。

    <html>
  <head>
    <title>Example HTML Page</title>
  </head>
  <body>
    <noscript>
      <p>本页面需要浏览器支持（启用）JavaScript.</p>
    </noscript>
  </body>
</html>

    # 第三章 基本概念

    # 3.1 语法

    ECMAScript 的语法大量借鉴了 C 及其他类 C 语言，如 Java 和 Perl，并且更加宽松

    # 3.1.1 区分大小写

    变量、函数名、操作符都区分大小写，变量 test 和 Test 表示两个不同变量

    # 3.1.2 标识符（Identifier）

    标识符指的是变量、函数、属性的名字，规则如下：

    • 第一个字符必须是字母、下划线（_）或美元符号（$）
    • 其他字符可以是字母、下划线、美元符号和或数字（与第一个字符不同的是可以用数字）
    • 按照惯例，采取小驼峰命名
    • 不能使用关键字、保留字、true、false、null 作为标识符
    • 不推荐使用 ASCII 以外的字母字符

    # 3.1.3 注释

    // 单行注释

/*
 * 这是一个多行（块级）注释
 * 采取这种格式提高可读性
 */

    # 3.1.4 严格模式

    在顶部添加代码

    'use strict';

    或者在函数内部最上方添加，指定函数在严格模式下执行

    function doSomething() {
  'use strict';
  // 函数体
}

    注：ES6 的模块自动采用严格模式

    # 3.1.5 语句

    • 建议语句结尾使用分号
    • 建议条件控制语句，始终使用代码块

      分号的问题看团队或个人的偏好，很多工具支持自动化添加/去除分号 很多成熟的 JavaScript 代码库，采用不使用分号的代码风格，比如 vue，做到代码风格的统一即可

    # 3.2 关键字和保留字

    关键字和保留字不能用作标识符，这里就不列举了

    # 3.3 变量

    使用 var 关键字定义变量

    var message; // 这种定义方式，message保存的值为undefined

    不推荐改变变量类型

    var message = 'hi';
message = 100; // 有效，但不推荐

    省略 var 操作符，直接赋值一个未定义的变量，将创建一个全局变量（不推荐）

    function test() {
  message = 'hi'; // 全局变量
}
test();
alert(message); // "hi"

    一条语句定义多个变量，使用逗号分隔

    var message = 'hi',
  found = false,
  age = 29;

    # 3.4 数据类型

    1 种基本上数据类型： Undefined、Null、Boolean、Number、String 1 种复杂数据类型：Object ES6 新增：Symbol

    # 3.4.1 typeof 操作符

    typeof 返回值：

    • "undefined" —— 值未定义
    • "boolean" —— 值是布尔值
    • "string" —— 值是字符串
    • "number" —— 值是数值
    • "object" —— 值是对象或 null
    • "function" —— 只是函数

    注意：

    1. 数组、正则表达式、null 的返回值为"object"
    2. typeof 是一个操作符而不是函数，可以使用括号但不是必需的，如 typeof (message)  或 typeof 95

    # 3.4.2 Undefined 类型

    • Undefined 类型只有一个值，即 undefined
    • 使用 var 声明变量但未对其初始化时，这个变量的值就是 undefined（let 同理）
    • 没有必要显示的使用 undefined 初始化变量，比如 var message = undefined
    • 使用 typeof 检测未声明的变量，值是 undefined
    • 一般情况下，建议始终都在声明变量时显示的初始化

    # 3.4.3 Null 类型

    • Null 类型只有一个值 null
    • 如果定义的变量在将来用于保存对象，可以初始化为 null
    • undefined == null -> true

    # 3.4.4 Boolean 类型

    • Boolean 值区分大小写，只有 true 和 false 是有效值
    • 使用 Boolean 可以将一个值转换为对应的 Boolean 值
    • 流程控制语句中，比如 if 语句，自动执行 Boolean 转换
      数据类型 转换为 true 的值 转换为 false 的值
      Boolean true false
      String 任何非空字符串 ""(空字符串)
      Number 任何非零数字值（包括无穷大） 0 和 NaN
      Object 任何对象 null
      Undefined 无 undefined

    # 3.4.5 Number 类型

    ECMAScript 中的 Number 类型使用 IEEE754 格式来表示整数和浮点数值（双精度数值）

    # 整数
    var intNum = 55; //十进制整数

var octalNum1 = 070; // 八进制的56
var octalNum1 = 079; // 无效的八进制数值，解析为79
var octalNum1 = 08; // 无效的八进制数值，解析为8

var hexNum1 = 0xa; // 十六进制的10
var hexNum2 = 0x1f; // 十六进制的31

    注：JavaScript 可以保存正零(+0)和负零(-0)。正零和负零被认为相等

    # 浮点数值
    var floatNum1 = 1.1;
var floatNum2 = 0.1;
var floatNum3 = 0.1; // 有效但不推荐

// 由于保存浮点数值需要的内存空间是保存整数值的两倍
// 因此ECMAScript会不失时机的将浮点数值转换为整数值
var floatNum4 = 1; // 解析为1
var floatNum5 = 10.0; // 解析为10

// e表示法（科学计数法）
var floatNum6 = 3.125e7;
// 默认情况下，ECMAScript会将小数点后面带有6个零以上的浮点数转换为e表示法
var floatNum7 = 0.0000003; //3e-7

    另外，值得注意的是浮点数的精度问题，例如，0.1 加 0.2 的结果不是 0.3 这是使用基于 IEEE754 数值的浮点计算的通病

    # 数值范围
    • 由于内存限制，ECMAScript 并不能保存世界上的所有数值
    • Number.MIN_VALUE 保存了最小数值， Number.MAX_VALUE  保存了最大数值
    • 超出范围的部分会被转换成正或负的 Infinity 值，访问 Number.NEGATIVE_INFINITY  和 Number.POSITIVE_INFINITY  也可以得到负和正的 Infinity 值
    • 使用 isFinite()  函数判断一个数值是不是有穷的（介于最大和最小值之间）
    #
    # NaN
    • NaN 是一个特殊的数值，用于表示本来要返回数值的操作数未返回数值的情况
    • NaN 与任何值都不相等，包括 NaN 本身，可以用 isNaN()  函数判断
    // isNaN会尝试将一个值转换为数值，不能转换为数值的情况会返回true
NaN == NaN; // false
isNaN(NaN); // true
isNaN(10); // false
isNaN('10'); // false
isNaN('blue'); // true(不能转换为数值)
isNaN(true); // false
    #
    # 数值转换

    有 3 个函数可以把非数值转换为数值，Number(), parseInt(), parseFloat() Number()函数转换规则如下

    • true 和 false 将分别转换为 1 和 0
    • 数字值返回数字本身
    • null 返回 0
    • undefined 返回 NaN
    • 字符串
      • 字符串只包含数字（包括正负号），转换为十进制数，注意"011"会变成 11，前导 0 忽略
      • 字符串包含有效的浮点形式，转换为对应的浮点数值
