    React Hooks 实践总结

    kuailoy 3/6/2020 reactjavascript

    # 前提条件

    要使用 Hooks，确保 React 版本在 16.8 以上

    在文件头部引入你要使用的 Hooks 即可，例如

    import React, { useState, useEffect } from 'react';

function Example() {
  const [state, setState] = useState(0);
  useEffect(() => {
    // ...
  });
  // ...
}

    # 常用 hooks

    下面介绍一些常用的 hooks，并且列出一些开发中常见的场景下比较好的实践

    # useState

    const [state, setState] = useState(initialState);

    • 返回一个状态值, 一个相应的更新函数

    • 初次渲染时, state 值为initailState值

    • setState 函数接收一个新的 state 值作为参数, 并触发 re-render

      注意: initailState 只会在第一次 render 时被使用, 随后的 render 会忽略它

    # 场景 1: 通过计算得到一个初始值

    如果子组件的 state 是根据 props 计算而来，这时可以给 useState 传入一个函数，返回计算后的结果

    返回值将作为 state 的初始值

    const [state, setState] = useState(() => {
  const initailState = computedInitialState(props);
  return initailState;
});

    # 场景 2: 基于 prevState 进行状态更新

    如果新的 state 是基于之前的 state 计算得来的, 可以给 setState 传入一个函数，类组件也有类似用法

    function Counter({ initialCount }) {
  const [count, setCount] = useState(initialCount);
  return (
    <>
      Count: {count}
      {/* 传入new state值 */}
      <button onClick={() => setCount(initialCount)}>Reset</button>
      {/* 传入函数 */}
      <button onClick={() => setCount((prevCount) => prevCount - 1)}>-</button>
      <button onClick={() => setCount((prevCount) => prevCount + 1)}>+</button>
    </>
  );
}

    # 场景 3: 更新 object 类型的 state

    对于 object 类型的 state，每次更新都是整体更新，例如：

    // position是对象类型的state
const [position, setPosition] = useState({ left: 0, top: 0 });

useEffect(() => {
  function handleWindowMouseMove(e) {
    // 更新时新对象将对state整体覆盖
    setPosition({ left: e.pageX, top: e.pageY });
  }
  // ...

    与类组件不同的是, setPosition不会自动合并对象的更新, 应使用展开语法或 Object.assign 合并对象后，返回新对象

    如果只想对对象类型 state 的某些字段进行更新，必须手动进行合并

    // 注意！错误写法！
setPosition({ left: e.pageX });

// 正确写法
setPosition((prevState) => {
  return { ...prevState, left: e.pageX };
});

    如果 state 更新值与当前值相同(通过 Object.is 比较), 将不会触发 render 和 effect, 而使用上述方法(引用类型 state), 总会触发新的 render

    在使用 hooks 时，是应该像类组件一样，使用一个大的 object 作为 state，还是使用多个 state？两者都可以，官方给的建议是尽量使用多个，这样就不必每次更新都手动合并对象了，

    仅仅将密切联系的一组值作为一个 state 是更好的选择，比如 上面例子中的position: { left: 0, top: 0 }

    # useEffect

    类组件中，经常用到生命周期方法来执行副作用

    相应的，函数式组件中可以使用useEffect来执行副作用，语法如下：

    function Demo() {
  useEffect(() => {
    // do something
  });
  // ...
}

    注意：在函数式组件的函数主体内不允许状态改变, 事件订阅, 定时器, 日志记录等副作用, 这么做可能会会导致 bug 或 UI 不连贯

    # 场景 1: 执行副作用

    // 类组件生命周期方法
class Example extends Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0,
    };
  }

  componentDidMount() {
    document.title = `You clicked ${this.state.count} times.`;
  }

  componentDidUpdate(prevProps, prevState) {
    document.title = `You clicked ${this.state.count} times.`;
  }

  render() {
    console.log('render');
    return (
      <div>
        <p>You clicked {this.state.count} times.</p>
        <button onClick={() => this.setState({ count: this.state.count + 1 })}>Click me</button>
      </div>
    );
  }
}

    // 使用 useEffect hooks
function Example(props) {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `You clicked ${count.value} times`;
  });

  return (
    <div>
      <p>You clicked {count.value} times.</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}

    # 场景 2: 订阅和清除订阅

    在 useEffect 传入的函数中，返回一个函数，这个函数将在组件 unmount 前执行

    其作用与componentWillUnmount类似

    // 类组件中使用生命周期方法
componentDidMount() {
  const subscription = props.source.subscribe();
}

componentWillUnmount() {
  subscription.unsubscribe();
}

// 使用 hooks
useEffect(() => {
  const subscription = props.source.subscribe();
  return () => {
    // 清除订阅
    subscription.unsubscribe();
  };
});

    # 场景 3: 给 effects 增加触发条件

    默认情况下, 每次 render 完成后都会触发 effects， 相当于在 componentDidMount、componentDidUpdate 都会触发

    在上面的例子中, 每次更新都会生成一个新的订阅, 我们可以通过下面的模式, 添加一个触发条件

    // 仅在props.source改变时生成一个新的订阅
useEffect(
  () => {
    const subscription = props.source.subscribe();
    return () => {
      // 清除订阅
      subscription.unsubscribe();
    };
  },
  [prop.source], // 给useEffect传入第二个参数
);

    如果只想在组件挂载时执行一次，即类似于 componentDidMount 的用法，可以给第二个参数传递一个空数组

    // 仅在组件挂载完成后执行一次
useEffet(
  () => {
    // do somthing...
  },
  [], // 传递一个空数组
);

    # 场景 4: 使用多个 useEffect

    在使用 class 声明组件时，经常会遇到这种情况，在生命周期函数中写了很多不相关的逻辑，代码看起来很乱

    而使用 useEffect 这个 hooks 时，可以把不相关的逻辑分开来写，例如：

    // class组件的用法
componentDidMount() {
  document.title = `You clicked ${this.state.count} times`;
  this.props.source.subscribe();
}
componentDidUpdate() {
  document.title = `You clicked ${this.state.count} times`;
}
componentWillUnmount() {
  this.props.source.unsubscribe()
}

    上面例子中，修改 title 需要在componentDidMount和componentDidUpdate中使用，而订阅和取消订阅需要在componentDidMount和componentWillUnmount中调用，在 hooks 中怎么使用 useEffect 来区分处理呢？

    很简单，使用多个 useEffect 把不同的逻辑分开来写就可以了

    function Example(props) {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `You clicked ${count} times`;
  });

  useEffect(() => {
    const subscription = props.source.subscribe();
    return () => {
      // Clean up the subscription
      subscription.unsubscribe();
    };
  });
}

    # 场景 5: 请求数据

    关于 hooks 请求数据，官方给出了一段比较完整的 demo：在输入文本同时获取搜索数据

    import React, { useState, useEffect } from 'react';
import ReactDOM from 'react-dom';
import axios from 'axios';

function SearchResults() {
  const [data, setData] = useState({ hits: [] });
  const [query, setQuery] = useState('react');

  // 在useEffect中进行数据请求，并且给出执行条件限制：修改query
  useEffect(() => {
    let ignore = false;

    async function fetchData() {
      const result = await axios('https://hn.algolia.com/api/v1/search?query=' + query);
      if (!ignore) setData(result.data);
    }

    fetchData();
    return () => {
      ignore = true;
    };
  }, [query]);

  return (
    <>
      <input value={query} onChange={(e) => setQuery(e.target.value)} />
      <ul>
        {data.hits.map((item) => (
          <li key={item.objectID}>
            <a href={item.url}>{item.title}</a>
          </li>
        ))}
      </ul>
    </>
  );
}

const rootElement = document.getElementById('root');
ReactDOM.render(<SearchResults />, rootElement);

    # useRef

    const refContainer = useRef(initialValue);

    通过 useRef 可以创建一个容器，并在.current属性中保存一个可变值

    initialValue将作为refContainer.current的初始值

    # 场景 1: 使用 DOM

    在 jsx 语法中，给标签设置 ref 属性，例如<div ref={myref}>

    这时 ref 容器的.current值将会与该 dom 节点绑定

    function TextInputWithFocusButton() {
  const inputEl = useRef(null);
  const onButtonClick = () => {
    // `current` 指向挂载后的input元素
    inputEl.current.focus();
  };
  return (
    <>
      <input ref={inputEl} type="text" />
      <button onClick={onButtonClick}>Focus the input</button>
    </>
  );
}

    # 场景 2: 实例属性

    class 组件中，经常需要用到实例属性，例如保存一组数据 this.data = {a: 1, b: 2}

    函数式组件中，如果直接在函数体内声明一个变量，在 re-render 时该变量值将被重置

    function Example(props) {
  // 注意！这里是错误写法，每次re-render会将变量重置
  let data = null;

  useEffect(async () => {
    // 挂载时获取数据
    const newData = await props.getData();
    data = newData;
  }, []);

  const getPrevData = () => data; // 组件更新后，data重置为null，数据丢失
  // ...
}

    正确的做法是，使用 useRef 创建一个容器，它的.current会保存一个类似实例变量的值

    这就保证了该值在函数组件 re-render 时不会改变，并且可以在整个函数内部被访问到

    function Example(props) {
  // 正确用法
  const dataRef = useRef();

  useEffect(async () => {
    const newData = await props.getData();
    dataRef.current = newData;
  }, []);

  const getPrevData = () => dataRef.current;
  // ...
}

    # 其他场景

    # 1. 如何实现 shouldComponentUpdate

    可以使用React.memo包裹一个函数式组件，它会在组件每次更新前对它的 props 进行浅比较

    React.memo并不是一个 Hook，其作用等同于类声明组件的pureComponent

    const Button = React.memo((props) => {
  // your component
});

    React.memo不比较 state，因为函数式组件不像类声明组件那样仅有一个 state，但是你可以通过React.memo包裹子组件，使子组件保持 “纯净”

    # 2. 如何使用 Context

    使用useContext这个 Hook，可以很方便的获取到 context，当然，前提是按照 React 的 context API 设置了 context

    function ThemedButton() {
  const theme = useContext(ThemeContext);

  return <button style={{ background: theme.background, color: theme.foreground }}>I am styled by theme context!</button>;
}

    # 一点思考

    # Why Hooks

    为什么要花时间学习和使用 Hooks？

    先看看官方给出的答案：

    # 1. 组件间的状态逻辑难以复用

    在传统的类声明组件中，没有提供一个具体的方法去实现组件间相同状态逻辑的复用，虽然可以通过render props或是高阶组件解决这个问题，但是你必须重新组织你的组件，并且代码会变的更加复杂和难以维护，很容易陷入“嵌套地狱”

    在 Hooks 中，可以使用自定义 Hooks 更简单的解决这个问题

    # 2. 大型组件复杂而难以理解

    在生命周期函数中，比如componentDidMount，componentDidUpdate, componentWillUnmount中，混杂着数据请求和各种逻辑不相关的副作用，使组件臃肿和难以理解。很多人使用第三方状态管理库如redux,mobx解决这个问题

    但是，状态管理库带来了更多的抽象概念，开发时需要在不同文件之间来回跳转，并且使组件复用变得更加困难

    Hooks 可以让你根据不同的逻辑对组件进行分割，必要时也可以通过reducer来管理本地状态

    # 3. 类让人困惑

    由于 Javascript 的原因，使用类声明语法无时无刻不在考虑this的问题，要记得绑定每个事件处理函数，还要在冗余的语法中挣扎...

    而 Hooks 可以让你在不使用类的情况下使用 React 的各种特性

    再来说说自己实际使用的感受：

    最直观的感受是，组件的语法更加简洁和便于维护，而且不用考虑类式声明中 this 的问题，组件内部 state 和方法都可以直接用变量来表示。

    useEffect确实提供了简洁性和便利性，但是实际使用中感觉语义化不如生命周期函数那么明确；另一方面，使用多个useEffect符合函数式编程的单一性原则，整体来看我认为是一个好的实践。

    # Hooks 与状态管理库结合

    虽然 Hooks 允许不同逻辑的组件分割，但并没有提供一个良好的实践规范，当组件的维护的状态很多时，仍要考虑怎么去进行逻辑和代码结构的划分，并且 props 的层层传递仍然是一个令人头疼的问题

    我认为目前有 3 个方向可以考虑：

    1. 总结一套组件分割的方案的良好实践
    2. 使用useReducer, useContext相结合，实现一套简单的类似 redux 状态管理方案，而直接引入 Redux 就稍显冗余了
    3. 目前现有的一些库，已经实现 hooks 结合 mobx 或 redux 的状态管理方案，例如mobx-react-lite (opens new window)，可以拿来用
